## 1. Phase 7F-A — Zone J PDF pagination (2026-05-08 13:30 UTC)

### Trigger
Varun screenshot in #zuildup-quotation-builder showing PDF rendering Zone J as a separate-looking table with duplicated `<thead>`/`<tfoot>` AFTER the Construction Total. HTML preview was correct.

### Diagnosis
- Zone J was added via Custom Additional Charge ("Steel Extra for 5.5-6kg avg" ₹5,21,135).
- Bug was PDF-only, HTML preview rendered correctly.
- Root cause: print pagination — Chrome auto-repeats `<thead>` and `<tfoot>` when a table breaks across pages (CSS spec: `display: table-header-group` / `table-footer-group`). Since cost table shared the `.calc-table` class with Area Calc tables (which legitimately need to break), the repeat behavior wasn't suppressed for the cost table.

### Fix (CSS-only, scoped via new class)
```js
// quote.js line 4111
<table class="calc-table cost-calc-table">  // added cost-calc-table

// quote.js line ~3478 (CSS print block)
.cost-calc-table { break-inside: avoid; page-break-inside: avoid; }
.cost-calc-table tfoot { break-inside: avoid; page-break-inside: avoid; }
```

Avoided modifying shared `.calc-table` to preserve Phase 7B-5 area-calc page-break hygiene. Cost table is realistically ≤10-11 rows; small white gap better than split with duplicated tfoot.

### Verification
- Tests: 174 passed
- Cloud Run revision: `zuildup-quotes-00028-r68`
- MD5 parity: `18ae9872568a8804a40d164b2855a220`
- PDF probed via puppeteer-core + headless Chrome + PyMuPDF: Cost header + Zone J + Sub-total + Construction Total all on same page (page 6).

### Commits
- `7b695ed` Phase 7F-A: PDF — keep Cost Calculation table on one page (Zone J fix)
- `513d7bb` docs: Phase 7F-A note

### Lesson learned
Don't share class names between tables that have different pagination needs. The cost table's "must stay together" requirement was at odds with Area Calc tables' "must break naturally" need.

## 2. Phase 7F-B — Blank pages elimination (2026-05-08 14:00–14:26 UTC)

### Trigger
Varun's actual client PDF (`/tmp/varun_actual.pdf`, 20 pages) had:
1. **Cost Calc heading on page 6, table on page 7, page 6 nearly blank** — 7F-A's `break-inside: avoid` pushed the table to its own page when it didn't fit, leaving heading orphaned.
2. **Area Calc orphan page 4** with just "Continued on next page →" line + footer.

7F-A had moved the Zone J symptom but the architectural problem remained: any `<section.pg>` that exceeds 297mm produces a partial 2nd physical page with the absolute-positioned `pg-foot` pinned to the bottom = orphan.

### Three-commit fix chain

#### Commit 1: `4748261` — Cost totals tfoot→tbody + Continued line gap
- Moved Sub-total / Lift Machine / Construction Total rows from `<tfoot>` into last `<tbody>` rows with classes `cost-totals-sub` / `cost-totals-grand`.
- Why: `<tfoot>` auto-repeats on every paginated page (`display: table-footer-group`), causing duplicate totals. Tbody rows with `break-before: avoid` stay glued to last zone, appearing exactly once.
- Replaced 7F-A "force on one page" CSS with new rules: `break-before: avoid-page` on totals rows + restored styling.
- Changed area calc "Continued" line from `margin-top: auto` to `margin-top: 6mm`.

#### Commit 2: `38ce86b` — splitPivot tightened
- Area Calc page-split logic chooses 'A' / 'AB' / 'ABC' based on row counts. Budget: page 1 zone-table region fits ~9 rows max.
- Was: `ab <= 9` → for dense quotes (basement+lift) this overflowed the page.
- Now: `ab <= 6` → forces 'A' pivot more aggressively, ensures content fits in 297mm.

#### Commit 3: `cea2aad` — floor summary padding trimmed
- Saved 14–17mm of vertical space in floor summary table.
- Brings Area Calc section under 297mm threshold, preventing overflow → orphan page.

### Verification
- Headless Chrome `getBoundingClientRect()` measurement confirmed all sections ≤ 297mm
- PDF: 13 pages → 12 pages (no blanks)
- Cloud Run: deployed revision `zuildup-quotes-00031-6tc`
- MD5 parity: `fb993c0d9b187b27ce303e8e13196fdc`
- Tests: 174 passed

### Commits
- `4748261` Phase 7F-B: PDF — eliminate blank pages on cost & area calc
- `38ce86b` Phase 7F-B (follow-up): tighten Area Calc splitPivot
- `cea2aad` Phase 7F-B (3rd pass): trim floor-summary vertical padding
- `1f9e6c4` docs: Phase 7F-B note

### Lessons learned
1. **`<tfoot>` auto-repeats** on every paginated page (`display: table-footer-group`). For totals-must-appear-once: put rows in `<tbody>` with `break-before: avoid`.
2. **`position: absolute` `pg-foot` + section overflow = orphan page** with footer pinned. Real fix is to make section fit in 297mm, not patch the footer.
3. **Measure section heights FIRST** in print debugging. `getBoundingClientRect()` via headless Chrome before patching CSS. The 7F-A → 7F-B → 7F-B-pad → 7F-B-budget chain was avoidable with one early measurement pass.
4. Hit SOUL.md "3-commit rule" amber zone but each commit had measurement evidence, not blind patching.

## 3. Phase 7F-C — Cost page polish (2026-05-08 14:50–18:30 UTC)

### Trigger
Varun's screenshot of post-7F-B PDF flagged two remaining cosmetic issues:
1. **Page 5 description (lede) doesn't span full width** — looked narrow against the full-width cost table below it.
2. **Page 6 repeats `<thead>` (Zone | Area | Rate | Total) when cost table continues** — Varun: "no need to give the headers for the table again — it is the continuation of the same table, so it should be directly zone J (not the headers)."

The 7F-B blank-page fix had a side-effect: by allowing natural pagination (dropping `break-inside: avoid`), the browser's default `<thead>` auto-repeat behavior surfaced when the table did paginate.

### Diagnosis
- `p.lede` CSS at line 3384 had `max-width: 440px` — applied globally, including the cost page.
- `<thead>` defaults to `display: table-header-group`, which auto-repeats on every paginated page. To suppress: change to `display: table-row-group` (treats it like a normal tbody row group, renders ONCE).

### Fix (CSS-only, scoped via new class)
```js
// quote.js line ~4102
return `
<section class="pg cost-calc-page">  // added cost-calc-page class
  ...

// quote.js line ~3496 (CSS print block)
.cost-calc-page p.lede { max-width: none; }  // override 440px
.cost-calc-table thead { display: table-row-group; }  // disable auto-repeat
```

### Verification
- DOM probe via puppeteer-core:
  - `costLede.maxWidth: "none"` ✅ (was `440px`)
  - `costLede.width_px: 643` ✅ (full content width)
  - `costThead.display: "table-row-group"` ✅ (was `table-header-group`)
  - `hasClass_costCalcPage: true` on cost section ✅
- Cloud Run: deployed revision `zuildup-quotes-00032-pqr`
- MD5 parity HEAD == LIVE = `8fe516ca958a3f058787e2aa16a61380`
- Tests: 174 passed

### Commits
- `ec62679` Phase 7F-C: cost page — full-width lede + suppress thead-repeat on continuation

### Lessons learned
1. **CSS `display` values for tables are powerful**:
   - `table-header-group` (default for `<thead>`) → auto-repeats on every paginated page
   - `table-row-group` → renders ONCE (treats element like a normal `<tbody>` row group)
   - `table-footer-group` (default for `<tfoot>`) → also auto-repeats
2. **Always scope CSS overrides by class**, not by element. Adding `.cost-calc-page p.lede` lets the global 440px lede stay for narrative pages while only the cost page overrides it.
3. **Verify CSS overrides via getComputedStyle**, not just DOM inspection. The DOM probe pattern in `/tmp/zonej_verify5.js` checks both `getBoundingClientRect()` for layout AND `getComputedStyle()` for applied CSS values.
