## Bug

Symptom: on a fresh Structure Only quote, every spec row's Description column rendered blank on the right-side preview. Rate column rendered `—` (which is by design — structure rate is global ₹/sqft, not per-row).

Root cause: Phase 7H-B added the `_canDefault` gate at line 4616 of `app/quote.js`:
```js
const _canDefault = (state._isFreshQuote === true) || (row._isFresh === true);
```
The gate prevents loaded quotes from silently reverting user edits. But on save+reload, `_isFreshQuote` is forced to `false`. The only catalog-seed helper, `defaultRowsFor()` (line 903), returned rows WITHOUT `_isFresh: true` on them. So after save+reload of a structure-only quote, every catalog-default row hit the `desc=''` branch.

In full-package quotes the bug was masked because reps customise descriptions per zone (so `override.description` is set, the `userOverroteDesc` branch wins). In structure-only quotes the items are standard and rarely edited, so the bug was visible across 25+ rows.

## Fix

One line in `app/quote.js:914`:
```js
// before
return cat.map(it => ({ id: it.id, override: {} }));
// after
return cat.map(it => ({ id: it.id, override: {}, _isFresh: true }));
```

`_isFresh: true` is a PER-ROW flag. It does NOT violate the 7H-B invariant: any row with `override.description` set still wins via the `userOverroteDesc` branch — `_isFresh` only matters for rows that are not user-overridden.

## Why one helper is enough

Verified via grep — there is exactly ONE seed helper in `app/quote.js`:
- `defaultRowsFor(scope, opts)` at line 903 — called from 5 sites (1556, 1780, 1956, 1963, 3639) covering bootForm seed, build-type auto-flip to structure_only, scope toggle, reset, and one more reset path.

No `defaultSpecRowsForStructure`, no `defaultSpecRowsForPackage`, no tier-specific helpers. The hallucinating subagents invented those.
