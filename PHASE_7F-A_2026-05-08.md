# Phase 7F-A — 2026-05-08 (Zone J PDF fix, hot-shipped)

**Trigger:** Varun reported via #zuildup-quotation-builder that on a 200sqy royal quote with 5 additional charges (Elevation through Steel Extra), Zone J was rendering in a SEPARATE table below the first table's totals in the **PDF only** (HTML preview was correct), with the headers and totals duplicated. Steel Extra ₹5,21,135 LOOKED disconnected from the running totals (math was actually correct — sub-total already included it).

**Root cause:** Print pagination. The cost-calculation table grew tall enough on this quote to cross the A4 boundary. Chrome's print engine, when a `<table>` paginates, repeats `<thead>` on every new page (correct behaviour) AND renders `<tfoot>` at the bottom of every page fragment. With one row (Zone J) on page 2, this produced what looked like a duplicate table.

The Phase 7B-5 page-break hygiene block (line 3450 of `app/quote.js`) deliberately allowed `.calc-table` to break **between zones** because the **Area Calculation** page genuinely has too much content to fit on one page. But the **Cost Calculation** page reused the same `.calc-table` class and inherited that policy, when it actually wants the opposite behaviour.

**Fix:** CSS-only, scoped via a new `.cost-calc-table` class on the cost page table only.

```diff
+  .cost-calc-table { break-inside: avoid; page-break-inside: avoid; }
+  .cost-calc-table tfoot { break-before: avoid-page; page-break-before: avoid; break-inside: avoid; page-break-inside: avoid; }
+  .cost-calc-table tfoot tr { break-before: avoid-page; page-break-before: avoid; }

-  <table class="calc-table">
+  <table class="calc-table cost-calc-table">
```

The cost table is short (≤10–11 rows realistically), so allowing it to push entirely to the next page (with a slightly larger white gap on the previous page) is far better than splitting it. The Phase 7B-5 area-calc page-break hygiene is preserved — that table genuinely needs to break across pages and the existing per-row 'avoid' rules still apply via the original `.calc-table` selector.
