### Backend layer (`app/server.js`)

Two new endpoints inserted just before the `GET /` static fallback:

#### `POST /api/quote-edit`

```jsonc
// Request
{
  "userText": "set zone A to 3500 and add a lift",
  "state":    { /* full client quote state */ },
  "history":  [{ "role": "user|assistant", "text": "..." }] // last 6 turns
}

// Response (200)
{
  "patches": [
    { "op": "set", "path": "pricing.zoneARate", "value": 3500,
      "explanation": "Updated Zone A rate to ₹3,500/sqft" },
    { "op": "set", "path": "build.hasLift", "value": true,
      "explanation": "Added lift to the building" }
  ],
  "note": "Zone A rate increased and lift enabled",
  "source": "llm",
  "reqId": "qedit_mpcau81w_o7s7rs"
}

// Response (503) — when ANTHROPIC_API_KEY missing
{ "error": "LLM backend not configured…", "patches": [] }
```

- Builds compact state snapshot (skips full row catalog data; just `id` + `label` + `overrides`)
- System prompt embeds full patch-op whitelist + allowed paths
- Calls Anthropic Messages API directly (`https://api.anthropic.com/v1/messages`)
- Defense-in-depth: client validates each patch again before applying
- 45 s timeout; strips markdown fences if Sonnet wraps JSON; finds `{…}` boundary on prose-leak
- Logs every request + response to `/tmp/quote-edit-log.jsonl` and Firestore `quote_edit_logs` collection

#### `POST /api/quote-edit-feedback`

```jsonc
// Request
{ "reqId": "qedit_…", "action": "apply|reject|apply_fail|validation_fail",
  "patch": { … }, "explanation": "…", "userText": "…", "error": "…" }
// Response: { "ok": true }
```

Both dual-logged (JSONL + Firestore) for Phase 9 mining.

### Patch system (deterministic, no silent mutation)

**Three ops:**
- `set` — dotted path or `rows[<row_id>].override.<field>`
- `add_row` — by `item_id`
- `delete_row` — by `row_id`

**Path whitelist (enforced both client-side and via Sonnet system prompt):**
```
customer.{salutation,name,address}
build.{plotSqYards,breadth,coverage,buildType,floors,hasBasement,hasLift,hasWaterTank}
pricing.{costPerSqft,zoneARate,zoneBRate,zoneCRate,zoneDRate,basementRate,liftCost}
rows[<id>].override.{label,rate,rate_text,brands,description,location,category_label}
notes
scope
```

ANY patch outside whitelist throws `path not allowed: <path>` — no silent corruption.

**Type coercion:**
- Numeric paths (Rate, Cost, costPerSqft, liftCost, plotSqYards, breadth, coverage, floors) → `parseFloat`
- Boolean paths (hasBasement, hasLift, hasWaterTank) → `!!`

### UX flow (AI Edit)

1. Rep clicks 🤖 AI Edit → drawer slides in
2. Types prompt: "customer is mr arjun verma, plot in sector 82 noida, bump zone B to 1500"
3. `callBackendOrLocal()` POSTs to `/api/quote-edit` with last 6 history turns + state snapshot
4. Sonnet returns N patches
5. Each patch becomes a yellow diff card with before → after + explanation
6. Rep clicks per-card Apply / Reject, or bulk Apply-all / Reject-all
7. Apply → re-validates → mutates `state` → persists to localStorage/Firestore → page reload → real calc engine recomputes everything
8. Every action POSTs to `/api/quote-edit-feedback` for the learning loop

### Fallback behavior

If `/api/quote-edit` returns non-200 (including 503 "no key"), client silently falls back to a **local regex-based intent parser** built into `initAIChat()`. Handles:
- "rename customer to X"
- "address is Y"
- "set zone A to N" / "bump zone B to N" (multi-zone in one prompt)
- "set floors to N"
- "plot size N sqyd"
- "add/remove lift / basement"
- "change X to Y" → fuzzy brand swap on matching rows (capped at 5)

Local parser is the safety net for any LLM availability issue.

## Key learnings captured this phase

### FS sync ghosts
- `gdrive-mount` on the VM occasionally claims files don't exist for several seconds after a successful write
- **Workaround that works every time:** `sync && sleep 8–12` between write and read, OR write via Python heredoc with absolute paths
- The Edit tool is the worst-affected; Python file writes are robust

### Cloud Run deploy timing
- `gcloud builds list` lags reality — status can show WORKING for minutes after build is actually done
- **Reliable check:** `gcloud builds describe <id> --format="value(status)"`
- Deploys including the 139 KB catalog JSON take 4–8 min, not the usual 2–3
- Don't panic at 5-min mark

### Anthropic key forms (this confused me for 30 min)
- `sk-ant-api03-…` — **direct API key**, pay-per-token, works against `api.anthropic.com` directly
- `sk-ant-oat01-…` — **OAuth access token from Claude Max subscription**, only works via the local billing proxy at `127.0.0.1:18801`. `api.anthropic.com` rejects it with 401.
- For any production Cloud Run service that needs LLM access, you need `api03`.

### Three Rules (re-validated this session)
1. **Read docs first** — Anthropic Messages API shape, env-var update pattern, gcloud run flags
2. **Verify end-to-end on live URL** — 3-way MD5 + live POST with real prompts on prod
3. **3+ commits on same bug = stop, rethink** — N/A this session, no bug loops

## Hotfix — 2026-05-20 (commit c7bade8, revision 00056-sqb)

**Bug:** Quick Build button did nothing; AI Edit drawer opened but Send threw.

**Root cause:** `initWizard` and `initAIChat` IIFEs were appended at the bottom of `quote.js`, inside the main IIFE but **outside** the `bootForm()` function where `state`, `toast`, `openModal`, `closeModal` are declared. Every reference to those names threw `ReferenceError`. The error was caught silently by the click handler's outer try/catch (or by the browser's "error in onclick" silent path), so the UI just appeared dead.

**How Phase 8 verification missed it:** The original ship validated the backend with curl POSTs to `/api/quote-edit` and confirmed MD5 parity on the JS files. It did NOT click the buttons in a real browser. **This is exactly the "verify END-TO-END on live URL" failure mode in MEMORY.md.**

**Fix:** In `bootForm()`, right after `let state = loadState();`, expose state + helpers via `window.__qbState / __qbToast / __qbOpenModal / __qbCloseModal`. In both Phase 8 IIFEs, replace free-variable references with `window.__qb*`.

**Lesson for next Phase:** For any new top-level IIFE that references symbols declared inside `bootForm()`, either:
1. Move the IIFE inside `bootForm()`, or
2. Explicitly expose what it needs via the window object,
3. AND click-test in a real browser as part of the ship verification checklist — not just curl the API.
