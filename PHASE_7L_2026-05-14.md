## Fix Detail (Batch B+C+D)

### Issue 1 — Fonts (the messy one)

**Symptom:** PDF body text rendered as Courier; ₹ glyph missing.

**Root cause:** `index.html` (and the preview render flow) only linked Google Fonts via `<link href="https://fonts.googleapis.com/...">`. Cloud Run's print container (`google-chrome-stable` + `fonts-liberation` only — see `Dockerfile`) has no internet access during `--print-to-pdf`, so Chrome falls back to Liberation/Courier.

**Fix:** Inline real font files as base64 inside `@font-face` `src: url(data:font/woff2;base64,…)` declarations in `app/quote.js`, which is loaded into every PDF render. No new `/api/font` endpoint, no `inlineLocalAssetsAsync` patch — keeps the surface area small.

- Fonts downloaded: Inter v4 variable (full 100–900 axis, includes ₹ U+20B9 from Currency block), Fraunces 400 / 600 / 700 latin.
- `@font-face` blocks at `app/quote.js` ~lines 3746, 3751, 3754, 3759.
- Form preview (in-browser, not PDF) still uses the live Google Fonts CDN — that's fine, browser has internet.

**Verification pending (Varun):**
- Export a PDF
- Confirm body = Inter, headings = Fraunces (NOT Courier / Times)
- Confirm ₹ renders as ₹ (not as `□` or `Rs.`)

If Courier still appears, the inline `@font-face` isn't reaching Chrome at print time — next move would be to inject the same `@font-face` block into the preview-render HTML's inline `<style>` directly (rather than relying on quote.js being loaded into the print frame). Flag and dig.

### Issue 2 — Full-width divider under spec h1

**Where:** `app/quote.js`, scoped selector `.pg.pg-specs-flow h1.section` (~line 3792).
**What:** Added a 1px gold rule that spans the full content column (page width − padding), not just the text width. Other pages still use the original tighter rule via existing `h1.section` styles.

### Issue 3 — Form section reorder

**Where:** `app/quote.js` form template + `app/index.html` mirror.
**Order (final):** Customer → Build Config → Floor Summary → Area Overrides → Pricing → Per Line Item Override → Additional Charges → Specifications → Notes.
**Notable:** Extracted `<details id="item-rate-overrides">` into its own fieldset (`item-rate-overrides-fs`) so sales can collapse it independently.

### Issue 5 — Cost Calc single-page fit

**Where:** `app/quote.js` after ~line 3954.
**What:** Compact CSS overrides — tightened row height, reduced top/bottom margin, scaled down "Notes" rows. Aim: full Cost Calc must fit one A4 page.

### Issue 6 — Removed "Continued on next page →" on Area Calc

## Operational Lessons / Gotchas (so next session doesn't repeat them)

1. **FS race after `exec`:** Right after long bash blocks, `quotation-builder/` may not exist for ~5–10s. Always use absolute paths or `sleep 5-10` if a `cd` fails.
2. **`gcloud run deploy --source` can stall on `nohup`:** First attempt (PID 3553034) silently died after "Uploading sources" — no revision was created. Re-ran synchronously in foreground, succeeded. **If `nohup` + log file ends mid-progress, re-run synchronously.**
3. **Don't trust `gcloud builds list` for `--source` deploys:** It returned 0 items even though Cloud Build was running (`--source` uses Buildpacks under the hood, doesn't always appear in the same list).
4. **Live MD5 only meaningful at the served path:** `quote.js` is served at `/app/quote.js`, NOT `/quote.js`. `/quote.js` returns 405. Always check the actual `<script src="…">` in the live `index.html` first.
5. **`Glob` and `BashSession` tools are not available** in this environment. Use `Bash` with `find` / `ls` instead.
